# Excel ドキュメント生成 Skill

## 概要

既存ドキュメント（Markdown）またはコードベースから、Excel形式のドキュメントを自動生成するSkillです。

**自動判定**: ユーザーが「Excelで作成して」「Excel形式のドキュメント」などと言えば、Claudeが自動的にこのSkillを使用します。

---

## 🎯 使用方法

### 基本的な使い方

```
Cron処理のドキュメントをExcel形式で作成して
```

Claudeが自動的にこのSkillを起動し、以下のステップを実行します：
1. スタイル選択（A型/B型/H型）
2. 体裁の基準確認（サンプルExcelまたはデフォルト）
3. ソースの選択（Markdownまたはコードベース）
4. データ収集とコードベース検証
5. Excel生成
6. 検証と完了報告

---

## 📊 3つのスタイル

### A型（ビジネス文書スタイル）

**対象読者**: 経営層、管理職、非技術者

**特徴**:
- ✅ 敬語表現（「〜様」「〜いたします」）
- ✅ 物語的で読みやすい
- ✅ 画面区分が機能カテゴリ単位

**例**:
```
LIL社員様が作成した指導者宛てのメールを一斉送信いたします。
配信ステータスが配信予約で、配信予約時刻がコマンド実行の時刻になったメールが対象となります。
```

**使用シーン**: 社外提出、経営報告、概要説明

---

### B型（技術仕様書スタイル）

**対象読者**: 開発者、技術者

**特徴**:
- ✅ 簡潔な技術表現
- ✅ 構造化（改行で処理ステップを明示）
- ✅ テーブル名、カラム名を明記
- ✅ 画面区分が処理内容を具体的に記載

**例**:
```
LIL社員が作成した指導者宛てのメールを一斉配信します。
配信対象メール取得：mail_detail テーブルから配信予約済みメールを取得
配信先リスト取得：mail_history テーブルから配信先を取得
メール送信処理：各教室の指導者にメール送信
ステータス更新：送信成功(3)、送信失敗(4)、配信完了(2)
```

**使用シーン**: 日常的な技術リファレンス、トラブルシューティング、実装作業

---

### H型（ハイブリッドスタイル）

**対象読者**: 技術・非技術の混成チーム

**特徴**:
- ✅ A型の敬語表現 + B型の構造化
- ✅ 見やすい改行あり、空行なし
- ✅ テーブル名を括弧内に記載

**例**:
```
LIL社員様が作成した指導者宛てのメールを一斉配信いたします。
配信対象メール取得（mail_detail テーブルから配信予約済みメールを取得）、配信先リスト取得（mail_history テーブルから配信先を取得）、メール送信処理（各教室の指導者にメール送信）、ステータス更新（送信成功: 3、送信失敗: 4、配信完了: 2）、送信報告メール送信。
```

**使用シーン**: 技術者と非技術者の両方が参照する仕様書

---

## 🔧 体裁の基準

### デフォルト体裁

サンプルExcelがない場合、以下の体裁で生成します：

1. **タイトル行（Row 2）**: Meiryo 20pt、左揃え
2. **ヘッダー行（Row 4）**: 赤背景（#C22114）、白文字、太字、中央揃え、罫線
3. **データ行（Row 5以降）**: 罫線、左揃え、垂直中央、改行対応
4. **列幅**: 内容に応じて自動調整

### サンプルExcelがある場合

既存のサンプルExcelファイルの体裁を完全に解析し、以下を再現します：
- 列幅
- 罫線
- フォント（名前、サイズ、太字）
- 色（背景色、文字色）
- 配置（水平・垂直）
- 改行設定

## ✅ 品質保証

### 絶対原則

1. **推測厳禁**: すべてのデータはソースから取得
2. **省略禁止**: `...` や省略記号を使わない
3. **Code as Truth**: コードベースが唯一の真実
4. **Quality over Speed**: 質を最優先

### 検証項目

- [ ] データは完全か？（推測・省略なし）
- [ ] コードベース検証を実施したか？
- [ ] 体裁は正しく適用されているか？
- [ ] スタイルは適切か？
- [ ] ファイルサイズは適切か？

## 🚀 使用例

### 例1: Cron処理ドキュメント（技術仕様書スタイル）

**リクエスト**:
```
Cron処理のドキュメントをExcel形式で作成して
```

**Skillの動作**:
1. ✅ Skill自動起動（「Cron処理」「Excel」とマッチ）
2. ✅ スタイル選択プロンプト → ユーザーが「B」を選択
3. ✅ 体裁確認 → サンプルExcelのパスを提供または デフォルト体裁を使用
4. ✅ ソース選択 → `docs/specs/cron-jobs.md` を指定
5. ✅ データ収集 → 全12件のCron処理を抽出
6. ✅ コードベース検証 → Cronファイルの存在確認
7. ✅ Excel生成 → 技術仕様書スタイルで生成
8. ✅ 検証 → 完了報告

### 例2: API仕様書（ハイブリッドスタイル）

**リクエスト**:
```
API仕様書をExcelで作成して、技術者と非技術者の両方が読めるようにしたい
```

**Skillの動作**:
1. ✅ 「API仕様」「Excel」とマッチ
2. ✅ 「技術者と非技術者の両方」から、ハイブリッド型を推奨
3. ✅ 体裁確認 → デフォルト体裁を使用
4. ✅ ソース選択 → `docs/specs/api.md` または コードベースから生成
5. ✅ Excel生成 → ハイブリッドスタイルで生成

## 📝 テスト方法

### 1. Skillがロードされているか確認

Claude Codeを再起動後：
```
What Skills are available?
```

「excel-generation」が表示されれば成功。

### 2. Skillのトリガーテスト

以下のリクエストでSkillが自動起動するか確認：
```
Cron処理のドキュメントをExcel形式で作成して
API仕様書をExcelで作成
データベース設計書をExcel形式に変換
```

### 3. 生成されたExcelの品質確認

- データの完全性（推測・省略なし）
- 体裁の適切性（罫線、フォント、色）
- スタイルの正確性（A型/B型/H型）

## 🔍 トラブルシューティング

### Skillが自動起動しない

**原因**: descriptionとリクエストがマッチしていない

**対策**: 以下のキーワードを含めてリクエスト
- 「Excel」「Excel形式」「Excelで」
- 「Cron処理」「API仕様」「データベース設計」

### openpyxlがインストールされていない

```bash
pip install openpyxl
```

### 生成されたExcelが開けない

- ファイルサイズを確認
- Pythonスクリプトのエラーログを確認

## 📌 プロジェクトへの追加

このSkillはプロジェクトSkillとして `.claude/skills/excel-generation/` に配置されています。

**ファイル構成**:

```text
.claude/skills/excel-generation/
├── SKILL.md              # Claude向けSkill仕様
├── README.md             # このファイル（ユーザー向け説明）
├── TEST.md               # テスト方法
├── .gitignore            # Python関連ファイルを除外
└── scripts/              # 実装スクリプト（トークン効率化）
    ├── analyze_sample.py     # サンプルExcel解析
    ├── style_converter.py    # スタイル変換（A/B/H型）
    ├── excel_generator.py    # Excel生成エンジン
    └── main.py               # メインエントリーポイント
```

**チーム共有**:

```bash
git add .claude/skills/excel-generation/
git commit -m "Add Excel generation Skill (A型/B型/H型スタイル選択可能、完全実装)"
git push
```

チームメンバーがリポジトリをクローンすれば、自動的にこのSkillが利用可能になります。
